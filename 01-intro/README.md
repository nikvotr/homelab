# Introduction

This project is a hands-on approach to learning cybersecurity and system administration.

Instead of only solving CTF challenges, I decided to build a real environment where I can deploy, manage, and secure services.

## Objectives

The main goals of this homelab are:

- Understand how Linux systems work
- Learn networking basics in a real environment
- Practice deploying services using Docker
- Explore security concepts (both offensive and defensive)
- Document everything as a learning resource

## Why a Homelab?

A homelab allows me to:

- experiment safely without breaking real systems
- simulate real-world infrastructure
- gain practical experience instead of only theory
- understand how different components interact

## Learning Approach

This project is built step by step:

1. Install operating system
2. Configure storage
3. Set up networking and access (SSH)
4. Deploy services using Docker
5. Add monitoring and logging
6. Improve system security
7. Simulate attacks and analyze them

## Cybersecurity Focus

This homelab is not only about running services.

It is also used to:

- observe system behavior
- analyze logs
- understand attack surfaces
- practice system hardening

## Long-Term Vision

In the future, I want to expand this homelab with:

- vulnerable applications for practice
- monitoring and alerting systems
- network segmentation
- more advanced infrastructure tools

This project will evolve as my skills improve.
